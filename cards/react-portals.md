# Что такое React Portals и зачем они нужны?

## Ответ

**Portal** позволяет отрендерить дочерние элементы в другой DOM-узел, находящийся вне иерархии родительского компонента, — при этом в React-дереве (контекст, события) компонент остаётся на своём логическом месте.

```jsx
import { createPortal } from "react-dom";

function Modal({ children }) {
  return createPortal(
    <div className="modal">{children}</div>,
    document.getElementById("modal-root")
  );
}
```

**Зачем нужен:** модальные окна, тултипы, всплывающие уведомления часто должны визуально «пробить» родителей с `overflow: hidden` или `z-index`, чтобы не обрезаться и не теряться под другими слоями. Portal рендерит их прямо в `<body>` или отдельный контейнер, сохраняя при этом всплытие событий (`onClick` и т.д.) как будто компонент внутри обычного дерева.

Важно: portal меняет **место в DOM**, но не меняет место компонента в **React-дереве** — контекст (`useContext`) и обработчики событий продолжают работать как обычно.
