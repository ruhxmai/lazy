# Почему `[10, 1, 2].sort()` даёт `[1, 10, 2]`, а не `[1, 2, 10]`?

## Ответ

По умолчанию `Array.prototype.sort()` сравнивает элементы **как строки**, а не как числа. `"10"` лексикографически меньше `"2"`, потому что символ `'1'` меньше `'2'`.

```js
[10, 1, 2].sort();
// [1, 10, 2] — сортировка "по буквам"

[10, 1, 2].sort((a, b) => a - b);
// [1, 2, 10] — правильная числовая сортировка
```

Функция-компаратор `(a, b) => a - b` возвращает:
- отрицательное число → `a` идёт перед `b`
- положительное число → `b` идёт перед `a`
- `0` → порядок не меняется

**Ещё одна ловушка:** `sort()` (как и `reverse()`, `splice()`) **мутирует исходный массив**, в отличие от `map`/`filter`/`slice`.

```js
const original = [3, 1, 2];
const sorted = original.sort();
console.log(original); // [1, 2, 3] — изменился!

// Если исходный массив нельзя трогать:
const safelySorted = [...original].sort((a, b) => a - b);
```
