# Чем `==` отличается от `===` в JavaScript?

## Ответ

`===` (строгое равенство) сравнивает значения **без приведения типов**. `==` (нестрогое равенство) сначала приводит типы, а потом сравнивает — из-за этого результат часто удивляет.

```js
1 === '1';   // false — разные типы
1 == '1';    // true  — строка приводится к числу

null == undefined;  // true  — специальное правило
null === undefined; // false — разные типы

0 == false;   // true  — false приводится к 0
0 == '';      // true  — '' приводится к 0
'' == false;  // true

NaN === NaN;  // false — NaN не равен даже сам себе!
Number.isNaN(NaN); // true — правильный способ проверки
```

**Правило:** всегда используйте `===` и `!==`. Единственное осмысленное исключение — `value == null`, чтобы одной проверкой отловить и `null`, и `undefined`.

```js
if (value == null) {
  // сработает и для null, и для undefined
}
```
