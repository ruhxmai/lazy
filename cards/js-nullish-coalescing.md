# Чем `??` отличается от `||`?

## Ответ

Оба оператора подставляют значение по умолчанию, но по-разному решают, когда левая часть считается "пустой".

**`||`** (логическое ИЛИ) заменяет **любое** falsy-значение: `false`, `0`, `""`, `null`, `undefined`, `NaN`.

**`??`** (nullish coalescing) заменяет **только** `null` или `undefined` — остальные falsy-значения остаются как есть.

```js
const count = 0;

console.log(count || 10); // 10 — 0 считается "ложным"
console.log(count ?? 10); // 0  — 0 не null/undefined, оставляем как есть
```

Классическая ловушка — настройки или счётчики, где `0` или `""` являются **валидными** значениями, а не признаком отсутствия данных:

```js
function greet(name) {
  const a = name || "Гость";  // "" превратится в "Гость" — плохо, если "" валидное имя
  const b = name ?? "Гость";  // "" останется "" — заменится только null/undefined
}
```

`??` нельзя напрямую комбинировать с `&&`/`||` без скобок — JS выбросит `SyntaxError`:

```js
a || b ?? c;      // ❌ SyntaxError
(a || b) ?? c;    // ✅ ок
```
