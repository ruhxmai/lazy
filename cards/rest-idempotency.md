# Что такое идемпотентность HTTP-методов?

## Ответ

Метод **идемпотентен**, если повторный одинаковый запрос даёт тот же результат на сервере, что и один запрос (не создаёт побочных эффектов при повторе).

```
GET     — идемпотентен (просто чтение)
PUT     — идемпотентен (замена ресурса одними и теми же данными)
DELETE  — идемпотентен (удалили один раз — повтор ничего не меняет)
POST    — НЕ идемпотентен (каждый вызов может создать новый ресурс)
PATCH   — обычно НЕ идемпотентен (зависит от реализации)
```

Пример:

```
PUT /users/42 { "name": "Alex" }
-- Вызови хоть 10 раз — пользователь 42 всегда будет с name="Alex"

POST /users { "name": "Alex" }
-- Вызови 10 раз — получишь 10 разных пользователей с одинаковым именем
```

**Зачем это важно:** при сетевых сбоях клиент может безопасно повторить идемпотентный запрос (retry), не боясь дублирования данных. Для POST нужны отдельные механизмы — например, `Idempotency-Key` в заголовке.
