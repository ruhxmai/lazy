# Что такое идемпотентность HTTP-методов?

## Ответ

Метод **идемпотентен**, если повторный одинаковый запрос приводит к тому же состоянию сервера, что и один запрос — сколько раз ни отправляй, результат не меняется.

| Метод | Идемпотентен? | Почему |
|---|---|---|
| `GET` | ✅ Да | Только читает, ничего не меняет |
| `PUT` | ✅ Да | Полностью заменяет ресурс одними и теми же данными |
| `DELETE` | ✅ Да | После первого удаления повторный `DELETE` просто ничего не находит |
| `POST` | ❌ Нет | Каждый вызов обычно создаёт новую сущность |
| `PATCH` | ⚠️ Не гарантировано | Зависит от того, что именно патчится |

```
PUT /users/1 { "name": "Alex" }   → 1 раз или 10 раз, результат одинаков
POST /users  { "name": "Alex" }   → 10 раз = 10 новых пользователей
```

**Зачем это знать:** идемпотентные запросы безопасно повторять автоматически при обрыве сети (retry-логика в HTTP-клиентах). Повторять `POST` вслепую нельзя — можно создать дубликаты.
