# Как работает кэширование в HTTP?
## Ответ
Браузер сохраняет ответ сервера и не делает повторный запрос, пока кэш актуален.

Ключевые заголовки:
- `Cache-Control: max-age=3600` — кэшировать на 1 час
- `Cache-Control: no-cache` — всегда проверять актуальность у сервера
- `Cache-Control: no-store` — никогда не кэшировать (чувствительные данные)
- `ETag: "abc123"` — токен версии ресурса
- `If-None-Match: "abc123"` — браузер спрашивает: изменился ли ресурс?
- `304 Not Modified` — сервер отвечает: кэш актуален, тело не передаётся

```http
Cache-Control: public, max-age=86400
ETag: "v1.2.0"
```
