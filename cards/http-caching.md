# Как работает HTTP кэширование? Что такое Cache-Control и ETag?

## Ответ

Кэширование позволяет браузеру не делать повторный запрос, если данные ещё актуальны.

```http
# Сервер отвечает:
Cache-Control: max-age=3600
ETag: "abc123"

# При следующем запросе браузер отправляет:
If-None-Match: "abc123"

# Если ресурс не изменился:
HTTP/1.1 304 Not Modified   ← тело не передаётся, экономия трафика
```

- `max-age=N` — кэшировать N секунд
- `no-cache` — всегда проверять актуальность у сервера
- `no-store` — вообще не кэшировать
- `ETag` — «отпечаток» версии ресурса; браузер сверяет его при следующем запросе
