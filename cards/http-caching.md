# Как работает кеширование HTTP и какие заголовки за него отвечают?

## Ответ

Кеш управляется заголовком `Cache-Control`:

- `max-age=3600` — ресурс свежий 1 час, запрос на сервер не идёт
- `no-cache` — всегда проверять актуальность у сервера перед использованием
- `no-store` — не кешировать вообще (чувствительные данные)
- `public` — можно кешировать в CDN; `private` — только в браузере

`ETag` + `If-None-Match` — валидация: сервер возвращает `304 Not Modified`, если ресурс не изменился:

```http
GET /style.css HTTP/1.1
If-None-Match: "abc123"

HTTP/1.1 304 Not Modified
```

`304` не передаёт тело — экономит трафик.
