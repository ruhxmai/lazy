# Как работает специфичность (specificity) CSS-селекторов?

## Ответ

Специфичность определяет, **какое правило победит**, если несколько селекторов задают один и тот же стиль. Считается как вес из 4 категорий (от старшей к младшей):

| Категория | Примеры | Вес |
|---|---|---|
| Инлайн-стиль | `style="color: red"` | 1,0,0,0 |
| ID | `#header` | 0,1,0,0 |
| Класс, атрибут, псевдокласс | `.btn`, `[type="text"]`, `:hover` | 0,0,1,0 |
| Тег, псевдоэлемент | `div`, `::before` | 0,0,0,1 |

```css
#nav .btn:hover { color: blue; }  /* 0,1,1,1 — id + class + псевдокласс */
.container div.btn { color: red; } /* 0,0,1,2 — class + 2 тега */

/* Побеждает #nav .btn:hover — старшая категория (ID) весомее */
```

**Порядок сравнения:** сначала сравниваются старшие категории (id), и только при равенстве переходят к младшим. Большее число классов **не** перевешивает один ID.

`!important` перебивает любую специфичность — используйте как крайний случай, иначе следующий `!important` будет ломать предыдущий.

При равной специфичности побеждает правило, **объявленное позже** в CSS.
