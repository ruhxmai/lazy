# Зачем нужен React.Fragment (`<>...</>`)?

## Ответ

JSX-компонент обязан возвращать **один корневой элемент**. Раньше для этого оборачивали разметку в лишний `<div>`, но это засоряет DOM и может ломать CSS (`flex`/`grid` ожидают прямых детей) и семантику HTML (например, `<tr>` внутри `<table>` не может содержать `<div>`).

**Fragment** группирует несколько элементов **без создания узла в DOM**.

```jsx
// Плохо — лишний div в разметке
function Row() {
  return (
    <div>
      <td>Имя</td>
      <td>Возраст</td>
    </div>
  );
}

// Хорошо — Fragment ничего не добавляет в DOM
function Row() {
  return (
    <>
      <td>Имя</td>
      <td>Возраст</td>
    </>
  );
}
```

Короткий синтаксис `<>...</>` не поддерживает атрибуты. Если нужен `key` (например, при рендере списка фрагментов), используйте полную форму:

```jsx
items.map(item => (
  <React.Fragment key={item.id}>
    <dt>{item.term}</dt>
    <dd>{item.definition}</dd>
  </React.Fragment>
));
```

**Итог:** Fragment решает проблему «обязательного одного корня» в JSX, не добавляя лишних DOM-узлов.
