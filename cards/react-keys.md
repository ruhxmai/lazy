# Зачем React нужны атрибуты key в списках?

## Ответ

`key` — специальный атрибут, который помогает React отслеживать элементы списка при **reconciliation** (согласовании виртуального DOM с реальным). Без стабильных ключей React может неправильно обновить DOM.

- Ключи должны быть **уникальными** среди элементов-соседей
- Используй стабильные `id`, а не индекс массива
- Индекс как ключ вызывает баги при сортировке, фильтрации, анимациях

```jsx
// Плохо — индекс как key (баги при изменении порядка)
items.map((item, index) => <li key={index}>{item.name}</li>)

// Хорошо — стабильный id
items.map(item => <li key={item.id}>{item.name}</li>)
```
