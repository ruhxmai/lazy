# Зачем React нужен атрибут key при рендере списков?

## Ответ

`key` — специальный атрибут, помогающий React понять, какой элемент изменился, добавился или удалился. Без него React перерисовывает весь список.

```jsx
// Плохо: нет key — React ругается в консоли
users.map(user => <li>{user.name}</li>);

// Плохо: key = index — ломается при сортировке и удалении
users.map((user, i) => <li key={i}>{user.name}</li>);

// Хорошо: key = уникальный стабильный id из данных
users.map(user => <li key={user.id}>{user.name}</li>);
```

**Правила key:**
- Уникален среди **соседних** элементов (не глобально)
- Стабилен — не меняется между рендерами
- Не доступен внутри компонента как prop (`props.key` === undefined)

При смене `key` React **уничтожает** старый компонент и монтирует новый — это можно использовать намеренно для сброса состояния.
