# Зачем React нужны key в списках?
## Ответ
`key` — уникальный идентификатор, который помогает React эффективно обновлять список: понимать, что изменилось, добавилось или удалилось.

```jsx
// Плохо: индекс как key при изменяемых списках
{items.map((item, index) => (
  <Item key={index} data={item} />
))}

// Хорошо: стабильный уникальный ID
{items.map((item) => (
  <Item key={item.id} data={item} />
))}
```

- Без `key` React может неправильно переиспользовать DOM-узлы
- `key` не передаётся в props компонента (не виден через `props.key`)
- Индекс подходит только если список не меняется и не фильтруется
