# Когда использовать CSS Grid, а когда Flexbox?

## Ответ

**Flexbox** — одномерная раскладка: элементы выстраиваются в **одну линию** (строку или столбец). Хорош для выравнивания и распределения пространства внутри компонента.

```css
/* Панель кнопок в одну строку с отступом между ними */
.toolbar {
  display: flex;
  gap: 8px;
  justify-content: space-between;
}
```

**Grid** — двумерная раскладка: строки **и** столбцы контролируются одновременно. Хорош для общей структуры страницы или сложных карточек.

```css
/* Layout страницы: шапка, сайдбар, контент, футер */
.page {
  display: grid;
  grid-template-columns: 240px 1fr;
  grid-template-areas:
    "sidebar header"
    "sidebar content"
    "sidebar footer";
}
```

| Задача | Flexbox | Grid |
|---|---|---|
| Навбар / список кнопок | ✅ | излишне |
| Центрирование одного элемента | ✅ (`align-items`, `justify-content`) | тоже можно, но проще Flexbox |
| Layout всей страницы | неудобно (много вложенных flex) | ✅ |
| Карточка с фиксированной сеткой (картинка+текст+кнопка) | сложно выровнять по 2 осям | ✅ |
| Контент "как получится", по одной оси | ✅ | избыточно |

**Правило:** Grid — для общей структуры (макро-layout), Flexbox — для выравнивания содержимого внутри блока (микро-layout). Их часто комбинируют: Grid снаружи, Flexbox внутри ячеек.
