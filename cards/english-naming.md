# Какие соглашения об именовании существуют в разработке?

## Ответ

Naming conventions — стандарты именования, принятые в разных языках и контекстах:

| Стиль | Пример | Где используется |
|-------|--------|------------------|
| camelCase | `getUserName` | JS переменные, функции |
| PascalCase | `UserProfile` | Классы, React-компоненты |
| snake_case | `user_name` | Python, SQL-колонки |
| UPPER_SNAKE | `MAX_RETRIES` | Константы |
| kebab-case | `user-profile` | CSS-классы, URL |

Полезные фразы:
- *"Use camelCase for function names"*
- *"Constants should be in SCREAMING_SNAKE_CASE"*
- *"React component names must start with a capital letter (PascalCase)"*
