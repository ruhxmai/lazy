# В чём разница между cookies, session и JWT для аутентификации?

## Ответ

| Механизм | Где хранится состояние | Как работает |
|----------|------------------------|--------------|
| **Cookie** | Браузер | Просто хранилище пары ключ-значение, автоматически отправляется с каждым запросом на тот же домен |
| **Session** | Сервер (память/Redis/БД) | Сервер хранит данные пользователя, а клиенту выдаёт только `session_id` в cookie — подход **stateful** |
| **JWT** | Клиент (в токене) | Токен содержит подписанные данные пользователя внутри себя, сервер ничего не хранит — подход **stateless** |

```
Cookie:  Set-Cookie: sessionId=abc123
Session: sessionId -> { userId: 42, role: "admin" }  (на сервере)
JWT:     eyJhbGciOi... (header.payload.signature) — сервер только проверяет подпись
```

Главный компромисс: сессии легко отозвать (удалил запись на сервере — пользователь разлогинен), но они не масштабируются без общего хранилища между серверами. JWT масштабируется легко, но токен нельзя отозвать до истечения срока действия без дополнительного blacklist-механизма.
