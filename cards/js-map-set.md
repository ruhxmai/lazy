# Чем Map и Set отличаются от обычных объектов и массивов?

## Ответ

**Map** — коллекция пар ключ→значение, где ключом может быть **любой тип** (в обычном объекте ключи всегда строки или Symbol). Порядок вставки гарантирован.

**Set** — коллекция **уникальных** значений. Дубликаты автоматически игнорируются.

```js
// Map — ключ любого типа
const map = new Map();
map.set('name', 'Alice');
map.set(42, 'ответ');
map.set({ id: 1 }, 'объект как ключ');

map.get('name'); // 'Alice'
map.size;        // 3
map.has(42);     // true

// Итерация
for (const [key, value] of map) {
  console.log(key, value);
}

// Set — только уникальные значения
const set = new Set([1, 2, 2, 3, 3]);
console.log([...set]); // [1, 2, 3]

// Популярный приём: убрать дубли из массива
const unique = [...new Set(['a', 'b', 'a'])]; // ['a', 'b']
```

- `Map` вместо `Object`: удобнее для динамических ключей и итерации
- `Set` вместо `Array`: когда нужна уникальность без ручных проверок
