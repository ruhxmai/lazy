# Чем `Map` и `Set` отличаются от обычного объекта и массива?

## Ответ

**`Map`** — коллекция пар ключ-значение, где ключом может быть **любой тип** (объект, функция, число), а не только строка, как у обычного объекта.

```js
const map = new Map();
const objKey = { id: 1 };

map.set(objKey, 'значение для объекта');
map.set('str', 'значение для строки');

map.get(objKey); // 'значение для объекта'
map.size;        // 2
```

**`Set`** — коллекция **уникальных** значений, автоматически отбрасывает дубликаты.

```js
const set = new Set([1, 2, 2, 3, 3, 3]);
[...set]; // [1, 2, 3]

set.add(4);
set.has(2); // true
set.delete(2);
```

**Когда предпочесть Map/Set объекту/массиву:**
- Ключи — не строки (например, DOM-элементы или объекты)
- Часто нужен `.size` (у объекта нет встроенного счётчика ключей)
- Нужна гарантия уникальности значений (Set) без ручной проверки `includes()`
- Важен порядок вставки и частые `add`/`delete` — у `Map`/`Set` они быстрее, чем у объекта с ручным управлением ключами

```js
// Убрать дубликаты из массива — частый junior-паттерн
const unique = [...new Set([1, 1, 2, 3, 3])]; // [1, 2, 3]
```
