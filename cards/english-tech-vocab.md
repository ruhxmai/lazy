# Как описать баг или задачу в трекере на английском?

## Ответ

Стандартные шаблоны для issue и задач:

```
# Заголовки (начинаются с глагола в инфинитиве)
[Bug]     Login button unresponsive on mobile
[Feature] Add pagination to user list
[Chore]   Upgrade React to v19

# Структура описания
Steps to reproduce:   — шаги воспроизведения
Expected behavior:    — ожидаемое поведение
Actual behavior:      — фактическое поведение
Acceptance criteria:  — критерии приёмки

# Частые глаголы в задачах
implement   — реализовать
refactor    — отрефакторить
investigate — исследовать
expose      — выставить наружу
deprecate   — объявить устаревшим
wire up     — подключить
```

Заголовок задачи — всегда инфинитив: *"Add user auth"*, не *"User auth adding"*.
