# Что такое нормализация базы данных?

## Ответ

**Нормализация** — процесс организации таблиц БД для устранения дублирования и аномалий при изменении данных.

| Форма | Правило |
|-------|---------|
| 1NF | Все значения атомарны, нет повторяющихся групп |
| 2NF | 1NF + каждый неключевой атрибут зависит от **всего** составного ключа |
| 3NF | 2NF + нет транзитивных зависимостей между неключевыми атрибутами |

```sql
-- ❌ Нарушение 1NF: несколько значений в одном поле
-- orders: id=1, products='book,pen'

-- ✅ Исправление (1NF): отдельная таблица
-- order_items: order_id=1, product='book'
-- order_items: order_id=1, product='pen'

-- ❌ Нарушение 3NF: city зависит от zip, а не от id
-- users: id, zip, city

-- ✅ Исправление: выносим в отдельную таблицу
-- zip_codes: zip, city
-- users: id, zip
```

Цель: каждый факт хранится **ровно в одном месте**.
