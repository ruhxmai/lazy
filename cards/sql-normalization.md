# Что такое нормализация БД? Объясни 1NF, 2NF, 3NF.
## Ответ
Нормализация — процесс организации таблиц для устранения дублирования данных и аномалий обновления.

- **1NF**: значения атомарны (нет массивов в ячейках), есть первичный ключ.
- **2NF**: соответствует 1NF + все неключевые атрибуты зависят от **всего** составного ключа.
- **3NF**: соответствует 2NF + нет транзитивных зависимостей (неключевой атрибут не зависит от другого неключевого).

```sql
-- Нарушение 3NF: city зависит от zip_code, а не от id
-- orders(id, customer_id, zip_code, city)  ← плохо

-- После нормализации:
-- orders(id, customer_id, zip_code)
-- zip_codes(zip_code, city)
SELECT o.id, z.city
FROM orders o
JOIN zip_codes z ON o.zip_code = z.zip_code;
```

Денормализация допустима ради производительности, но осознанно.
