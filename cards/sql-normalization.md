# Что такое нормализация БД и зачем она нужна?
## Ответ
Нормализация — приведение таблиц к стандартным формам для устранения дублирования и аномалий при изменении данных.

- **1NF** — значения атомарны (нет списков в ячейке), есть первичный ключ
- **2NF** — 1NF + нет частичных зависимостей от составного ключа
- **3NF** — 2NF + нет транзитивных зависимостей (A→B→C, где B не является ключом)

```sql
-- Нарушение 3NF: city зависит от zip, а не от user_id
-- users(user_id, name, zip, city)

-- Решение: разбить на таблицы
CREATE TABLE users (user_id INT, name TEXT, zip TEXT);
CREATE TABLE locations (zip TEXT PRIMARY KEY, city TEXT);
```

Нормализация уменьшает дублирование, но может замедлить чтение (нужны JOIN).
