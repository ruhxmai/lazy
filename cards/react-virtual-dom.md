# Что такое Virtual DOM в React и зачем он нужен?

## Ответ

Virtual DOM — это лёгкое JS-представление реального DOM в виде дерева обычных объектов. React хранит копию этого дерева в памяти и использует её для оптимизации обновлений интерфейса.

При изменении состояния React:
1. Строит новое Virtual DOM дерево
2. Сравнивает его со старым (**diffing**, алгоритм reconciliation)
3. Вычисляет минимальный набор изменений
4. Применяет только их к реальному DOM (**commit**)

```jsx
// Было
<div className="box">
  <p>Count: 1</p>
</div>

// Стало после setCount(2)
<div className="box">
  <p>Count: 2</p>
</div>

// React обновит только текстовый узел внутри <p>,
// а не пересоздаст весь <div>
```

Реальный DOM медленный для частых манипуляций, а работа с JS-объектами — быстрая. Virtual DOM позволяет batching (группировку) изменений и минимизирует число дорогих операций с настоящим DOM.
