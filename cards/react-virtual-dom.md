# Что такое Virtual DOM в React и зачем он нужен?

## Ответ

**Virtual DOM** — лёгкая JS-копия реального DOM в виде обычных объектов. React хранит два «снимка» дерева: до изменения и после, сравнивает их (**diffing**) и обновляет в реальном DOM только то, что реально изменилось.

```jsx
// До
<button className="btn">Save</button>

// После изменения state
<button className="btn active">Save</button>

// React не пересоздаёт весь узел —
// он находит разницу и меняет только className
```

**Почему это быстро:** прямые операции с реальным DOM (reflow, repaint) дорогие. Работа с обычными JS-объектами в памяти — дешёвая. React группирует изменения и применяет их одним пакетом (**batching**), минимизируя число обращений к настоящему DOM.

**Алгоритм диффинга (reconciliation) использует эвристики:**
- Разные типы тегов (`div` → `span`) → старое поддерево удаляется, строится новое с нуля
- Одинаковый тип → сравниваются только изменившиеся атрибуты/дети
- Списки сравниваются по `key`, а не по индексу — отсюда важность правильных ключей

```jsx
// Плохо: индекс как key ломает диффинг при пересортировке
items.map((item, i) => <Item key={i} {...item} />)

// Хорошо: стабильный уникальный id
items.map((item) => <Item key={item.id} {...item} />)
```

Virtual DOM — не панацея для скорости, а компромисс: он делает *предсказуемое* декларативное обновление UI ощутимо быстрее, чем наивная ручная перерисовка всего дерева при каждом изменении.
