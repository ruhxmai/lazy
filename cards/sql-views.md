# Что такое VIEW в SQL и зачем оно нужно?

## Ответ

`VIEW` — это **сохранённый SQL-запрос**, к которому можно обращаться как к обычной таблице. Сам по себе он не хранит данные — при каждом обращении выполняется исходный запрос.

```sql
CREATE VIEW active_users AS
SELECT id, name, email
FROM users
WHERE status = 'active';

-- Использование:
SELECT * FROM active_users WHERE name LIKE 'A%';
```

**Зачем нужен:**
- Скрывает сложность (можно спрятать JOIN из нескольких таблиц за простым именем)
- Ограничивает доступ (пользователю дают доступ к view, а не к чувствительным столбцам таблицы)
- Переиспользование часто повторяющихся запросов

**Важно:** обычный `VIEW` не ускоряет запрос — это просто «сохранённый алиас». Для кэширования результата на диск существует `MATERIALIZED VIEW` (нужно обновлять вручную или по расписанию через `REFRESH`).
