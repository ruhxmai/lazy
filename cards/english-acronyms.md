# Какие частые аббревиатуры используются в разработке (ASAP, WIP, LGTM и т.д.)?

## Ответ

Разработчики постоянно используют сокращения в чатах, PR и тикетах — незнание сбивает с толку даже при хорошем общем английском.

| Аббревиатура | Расшифровка | Значение |
|---|---|---|
| `ASAP` | As Soon As Possible | как можно скорее |
| `WIP` | Work In Progress | ещё не готово, в процессе |
| `LGTM` | Looks Good To Me | одобряю код-ревью |
| `PTAL` | Please Take A Look | пожалуйста, посмотри |
| `TBD` | To Be Determined | пока не решено |
| `TL;DR` | Too Long; Didn't Read | краткое содержание |
| `FYI` | For Your Information | к сведению |
| `ETA` | Estimated Time of Arrival | ожидаемый срок готовности |
| `POC` | Proof of Concept | пробный прототип |
| `AFAIK` | As Far As I Know | насколько мне известно |
| `IMO` / `IMHO` | In My (Humble) Opinion | по моему мнению |
| `N/A` | Not Applicable | неприменимо |
| `RFC` | Request For Comments | предложение для обсуждения |

```
PR comment: "LGTM, just one nit — see inline comment. PTAL after fixing."
Slack: "ETA on the deploy is EOD, still WIP on the migration script."
```

> `EOD` (End Of Day) и `EOW` (End Of Week) — ещё одна частая пара для дедлайнов.
