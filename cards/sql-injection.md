# Что такое SQL-инъекция и как от неё защититься?

## Ответ

**SQL-инъекция** — уязвимость, когда пользовательский ввод подставляется в SQL-запрос напрямую строкой, и атакующий может изменить смысл запроса.

```js
// УЯЗВИМО: строка собирается конкатенацией
const query = `SELECT * FROM users WHERE email = '${email}'`;
// Если email = "' OR '1'='1", запрос вернёт ВСЕХ пользователей
```

**Защита — параметризованные запросы (prepared statements):** значения передаются отдельно от текста запроса, драйвер БД сам их экранирует.

```js
// БЕЗОПАСНО: значение передаётся отдельным параметром
db.query('SELECT * FROM users WHERE email = ?', [email]);

// То же самое с именованными параметрами (PostgreSQL)
db.query('SELECT * FROM users WHERE email = $1', [email]);
```

Дополнительные меры: ORM (Prisma, Sequelize, TypeORM) по умолчанию использует параметризацию под капотом; принцип наименьших привилегий для пользователя БД; валидация и экранирование ввода как второй уровень защиты, но не замена параметризации.
