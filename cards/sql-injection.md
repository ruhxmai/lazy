# Что такое SQL-инъекция и как от неё защититься?

## Ответ

**SQL-инъекция** — атака, при которой пользовательский ввод подставляется в SQL-запрос напрямую (через конкатенацию строк) и меняет его логику.

```js
// ❌ Уязвимый код: ввод подставляется в запрос напрямую
const query = `SELECT * FROM users WHERE email = '${email}'`;

// Если email = "' OR '1'='1"
// Итоговый запрос:
// SELECT * FROM users WHERE email = '' OR '1'='1'
// → вернёт ВСЕХ пользователей, а не одного
```

Более опасный пример — удаление таблицы через `email = "'; DROP TABLE users; --"`.

**Защита — параметризованные запросы (prepared statements):** значения передаются отдельно от текста запроса, драйвер БД экранирует их сам.

```js
// ✅ Безопасно: значение передаётся как параметр, а не как часть строки
db.query('SELECT * FROM users WHERE email = ?', [email]);

// Node.js + pg
db.query('SELECT * FROM users WHERE email = $1', [email]);
```

**Правило:** никогда не собирать SQL через конкатенацию или шаблонные строки с пользовательским вводом. Современные ORM (Prisma, Sequelize, TypeORM) параметризуют запросы автоматически — но при написании «сырого» SQL это остаётся ответственностью разработчика.
