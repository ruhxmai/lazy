# Чем отличаются коды редиректов 301, 302, 307 и 308?

## Ответ

Все это коды 3xx, но они различаются по двум осям: **постоянный/временный** редирект и **сохраняется ли метод/тело запроса**.

| Код | Тип | Метод/тело |
|---|---|---|
| **301** Moved Permanently | постоянный | клиент **может** сменить `POST` на `GET` |
| **302** Found | временный | клиент **может** сменить `POST` на `GET` |
| **307** Temporary Redirect | временный | метод и тело **обязаны** сохраниться |
| **308** Permanent Redirect | постоянный | метод и тело **обязаны** сохраниться |

```http
HTTP/1.1 301 Moved Permanently
Location: https://example.com/new-path
```

**301 vs 308** — оба говорят "ресурс переехал навсегда", но исторически при 301 браузеры могли превратить `POST` в `GET` при переходе по редиректу. 308 гарантирует, что метод останется тем же.

**302 vs 307** — та же разница, только для временных редиректов.

Где это важно: если API-эндпоинт редиректит `POST`-запрос через 301/302, некоторые клиенты отправят второй запрос уже как `GET` — тело потеряется. Чтобы сохранить метод, нужно использовать 307/308.
