# Как грамотно оставить комментарий на code review по-английски?
## Ответ
Используй вежливые, конкретные формулировки — это профессиональный тон в большинстве команд.

| Ситуация | Фраза |
|---|---|
| Предложение | Could we consider using...? |
| Вопрос | What's the reason for...? |
| Найденная проблема | This might cause a bug when... |
| Похвала | Nice catch! / Great solution. |
| Блокер | This needs to be fixed before merge. |
| Мелкое замечание | Nit: rename this to `userCount`. |

**Nit** (nitpick) — незначительное замечание, которое не блокирует merge.

Всегда объясняй **почему** нужно изменить, а не только **что** — это делает ревью полезнее.
