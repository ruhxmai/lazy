# Жизненный цикл React-компонента

Каждый React-компонент проходит три фазы: **монтирование** (mount), **обновление** (update) и **размонтирование** (unmount). В функциональных компонентах управление жизненным циклом строится на хуке `useEffect`.

## Фазы жизненного цикла

### 1. Монтирование (Mount)
Компонент появляется в DOM. `useEffect` с пустым массивом зависимостей `[]` выполняется один раз сразу после первого рендера — идеальное место для загрузки данных или подписки на события.

```js
useEffect(() => {
  fetchUser(id); // выполняется один раз при монтировании
}, []);
```

### 2. Обновление (Update)
При изменении `props` или `state` React перерисовывает компонент. `useEffect` с зависимостями запускается каждый раз, когда одна из них изменяется.

```js
useEffect(() => {
  fetchUser(userId); // запускается при каждом изменении userId
}, [userId]);
```

### 3. Размонтирование (Unmount)
Компонент удаляется из DOM. Функция, возвращённая из `useEffect`, вызывается как **cleanup** — для отписки от событий, отмены запросов, очистки таймеров.

```js
useEffect(() => {
  const timer = setInterval(tick, 1000);
  return () => clearInterval(timer); // cleanup при размонтировании
}, []);
```

### Сравнение с классовыми компонентами

| Классовый метод | Эквивалент с хуками |
|---|---|
| `componentDidMount` | `useEffect(() => {…}, [])` |
| `componentDidUpdate` | `useEffect(() => {…}, [dep])` |
| `componentWillUnmount` | функция cleanup внутри `useEffect` |

## Схема

```mermaid
flowchart TD
    A([Создание компонента]) --> B[Первый рендер]
    B --> C[Монтирование в DOM]
    C --> D["useEffect с \"[]\" — mount"]
    D --> E{Props или state\nизменились?}
    E -- Да --> F[Повторный рендер]
    F --> G[Cleanup предыдущего эффекта]
    G --> H["useEffect с зависимостями — update"]
    H --> E
    E -- Нет --> I{Компонент\nудаляется?}
    I -- Нет --> E
    I -- Да --> J["Cleanup — unmount"]
    J --> K([Компонент удалён из DOM])
```
